## 📖 学习内容

今天系统学习了 React 的 useCallback 和 useMemo，理解了它们在性能优化中的应用场景。

### useCallback 深入理解

`useCallback` 返回一个记忆化的回调函数，只有在依赖项改变时才会更新。这对于传递给经过优化的子组件的回调函数非常有用。

```javascript
const memoizedCallback = useCallback(
  () => {
    doSomething(a, b);
  },
  [a, b],
);
```

**使用场景：**
- 传递给使用 `React.memo` 优化的子组件的回调
- 作为其他 Hook 的依赖项
- 避免在每次渲染时创建新的函数引用

### useMemo 性能优化

`useMemo` 返回一个记忆化的值，只有当依赖项改变时才重新计算。

```javascript
const memoizedValue = useMemo(
  () => computeExpensiveValue(a, b),
  [a, b]
);
```

**注意事项：**
- 不要过度使用，缓存本身也有开销
- 适用于计算昂贵的操作
- 可以用于避免不必要的渲染

### 实际案例

今天在项目中实现了一个优化：

```javascript
// 优化前 - 每次渲染都创建新数组
const sortedItems = items.sort((a, b) => a.name.localeCompare(b.name));

// 优化后 - 只在 items 改变时重新排序
const sortedItems = useMemo(
  () => [...items].sort((a, b) => a.name.localeCompare(b.name)),
  [items]
);
```

## 💡 收获与感悟

今天最大的收获是理解了性能优化需要"有的放矢"：

1. **不是所有函数都需要 useCallback** - 只有在需要保持引用稳定性时才使用
2. **useMemo 不是万能药** - 缓存有开销，简单计算不需要优化
3. **先测量，后优化** - 使用 React DevTools Profiler 找到真正的瓶颈

React 的优化哲学是"按需更新"，而不是"尽可能少更新"。理解这一点对于正确使用 Hooks 非常重要。
